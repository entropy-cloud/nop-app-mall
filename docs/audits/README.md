# Audits Guide

Use this directory for audit methodology and specialized audit records.

Start with `00-audit-execution-guide.md`.

Plan and closure audits are mandatory for created plans, but ordinary plan-audit and closure-audit evidence belongs in the plan by default.

Store audit records here only when the audit itself is specialized, complex, disputed, reusable, or useful for future replay. Examples include multi-dimensional audits, open-ended audits, design/architecture audits, document-code alignment audits, protected-area audits, historical baseline audits, or large findings ledgers.

Do not maintain a hand-written audit index. Discover audit records by filename, linked evidence from plans/logs, or search.

Recommended filenames when a separate audit file is justified:

- `2026-05-21-document-audit-user-management.md`
- `2026-05-21-multi-dimensional-audit-checkout-flow.md`
- `2026-05-21-doc-code-alignment-audit-order-list.md`
