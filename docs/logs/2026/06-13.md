### Session 2: sql-lib 规范修复 + NopFileRecord 测试修复

**sql-lib 规范修复（statistics-performance-plan 回归）**
- 根本原因：统计方法使用 raw `IJdbcTemplate` + `SQL.begin().param()` 而非 Nop sql-lib 规范
- 原因分析：sql-lib 的 `<source>` 使用 XPL 模板（`xpl:outputMode="sql"`），参数必须用 `${paramName}`（自动转 JDBC `?`），非 `:named`
- 之前错误使用 `:named`（MyBatis 风格），导致 H2 不识别
- `LitemallOrder.sql-lib.xml`: `:startDate` → `${startDate}`，添加 `<arg>` 声明，UPPER_SNAKE 别名 + `colNameCamelCase="true"`
- `LitemallOrderBizModel.java`: 移除 `IJdbcTemplate`，注入 `LitemallOrderMapper`，三个统计方法改为委托 mapper
- H2 兼容：`CAST(DELETED AS INT) = 0`、`CAST(ORDER_STATUS AS INT) = 101`
- 关键发现：`_dump/` 目录缓存了 XDEF validated 模型，Maven local repo 缓存了旧 JAR → 需 `mvn install` 刷新

**NopFileRecord 测试修复**
- 根本原因：`OrmFileComponent.onEntityFlush` → `DaoResourceFileStore.attachFile` 查找 NopFileRecord by fileId
- 测试数据中 `picUrl="/test/pic.jpg"` → `decodeFileId()` → `"pic"` → 找不到 NopFileRecord → 16 errors
- 修复：`TestLitemallOrderBizModel` 和 `TestLitemallCartBizModel` 采用 `createFileRecord()` 模式（参考 `TestLitemallAftersaleBizModel`）
- NopFileRecord PK 为 `FILE_ID`（单列），不同 bizObjName 需用不同 fileId

**修复的文件：**
- `app-mall-dao/.../sql/LitemallOrder.sql-lib.xml`: `${paramName}` 语法 + H2 CAST + UPPER_SNAKE 别名
- `app-mall-service/.../entity/LitemallOrderBizModel.java`: `IJdbcTemplate` → `LitemallOrderMapper`
- `app-mall-service/.../TestLitemallOrderBizModel.java`: `createFileRecord()` + 实际 picUrl URL
- `app-mall-service/.../TestLitemallCartBizModel.java`: `createFileRecord()` + 实际 picUrl URL
