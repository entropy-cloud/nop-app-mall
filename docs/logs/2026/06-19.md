### Phase 1 实施中触发的平台 bug 修复（nop-entropy）

`DataInitInitializer` 在 IoC 启动早期通过 `ioc:after` 链式触发（`BeanContainerImpl.getBean0:411-415` 遍历 `getNextBeans()`），`@PostConstruct init()` 调用时 `OrmTemplateImpl.sessionFactory` 字段尚未注入完成（IoC 循环依赖暴露半成品），`ormTemplate.runInSession(...)` 触发 `OrmSessionRegistry.get(null)` NPE。

修 `nop-entropy`：
- `nop-persistence/nop-orm/.../DataInitInitializer.java`：移除 `@PostConstruct` 注解（`init()` 方法保留）
- `nop-persistence/nop-orm/.../beans/orm-defaults.beans.xml`：`DataInitInitializer` bean 添加 `ioc:delay-method="init"`，`init()` 推迟到 IoC 完全启动后执行（`BeanContainerImpl.runDelayMethod:591-601`），循环依赖消失
- `mvn install -pl nop-persistence/nop-orm -am` 后重打 `app-mall-app` jar 验证

详见 `nop-entropy/ai-dev/logs/2026/06-19.md` 的对应条目（平台变更日志归 `nop-entropy/ai-dev/logs/`）。
