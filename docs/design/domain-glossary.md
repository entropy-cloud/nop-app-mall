# 领域词汇表

## 目的

统一 `nop-app-mall` 设计文档中的核心领域概念、标准中文译法和中英文对应关系。

本词汇表只收录跨多个设计文档重复出现、且容易产生歧义的核心术语。

## 使用规则

- 设计文档出现术语冲突时，以本词汇表为准。
- 如果某个概念已经有稳定中文译法，不要在不同文档中随意改写成新的近义词。
- 如果某个英文术语需要保留，应优先在首次出现时同时给出中文含义，再按文档风格决定后续是否继续保留英文。
- 本词汇表用于统一业务概念，不替代 `model/*.orm.xml` 中的字段、字典或状态码真相。

## 核心词汇

| 英文 | 中文标准译法 | 说明 |
|------|--------------|------|
| Mall user | 商城用户 | 面向前台浏览、下单、收货、退款等行为的终端用户 |
| Admin | 管理员 | 在分配职责范围内执行商品、订单、配置等运营动作的后台角色 |
| Super admin | 超级管理员 | 拥有全局后台管理权限的角色 |
| Category | 分类 | 用于组织商品导航和筛选的层级结构 |
| Brand | 品牌 | 商品所属的商业品牌 |
| Goods | 商品 | 面向用户展示的商品概念，通常对应 SPU 层 |
| SPU | SPU / 商品标准单元 | 用于表达商品公共信息的销售概念层 |
| SKU | SKU / 商品规格项 | 用户实际购买的具体规格组合 |
| Specification | 规格 | 用户可选择的变体维度，如颜色、尺码 |
| Attribute | 属性 | 仅用于描述展示、不直接影响购买的商品属性 |
| Cart | 购物车 | 用户下单前维护购买意图的临时容器 |
| Checkout | 结算 | 从购物车或直接购买进入订单提交前的确认过程 |
| Order | 订单 | 用户与商城之间的商业交易记录 |
| Order goods | 订单商品快照 | 下单时固化的商品与 SKU 信息 |
| Payment | 支付 | 用户完成订单付款的过程或状态 |
| Shipment | 发货 | 商家将商品交由配送链路处理的履约动作 |
| Receipt confirmation | 确认收货 | 用户或系统确认订单已完成收货的动作 |
| Refund | 退款 | 已支付订单在履约前后退回款项的过程 |
| After-sale | 售后 | 收货后或特定履约节点后的退款、退货退款等服务流程 |
| Coupon | 优惠券 | 基于规则提供优惠的营销对象 |
| Groupon | 团购 | 依赖多人参与条件成立后生效的营销交易模式 |
| Search history | 搜索历史 | 用户历史搜索记录 |
| Favorite / Collect | 收藏 | 用户主动标记商品或内容以便后续查看的行为 |
| Footprint | 浏览足迹 | 用户近期浏览记录 |
| Notice | 公告 | 管理员发布给用户或运营人员查看的系统/运营消息 |
| Notification | 通知 | 由业务事件触发的消息，如支付确认、发货提醒 |
| Region | 地区 | 用于地址选择和配送展示的参考地理层级数据 |
| Owner doc | Owner Doc / 归属文档 | 某个业务概念或规则的主维护文档 |

## 邻接说明

- 如果某个术语的业务语义发生变化，应同时更新相关 design owner doc 和本词汇表。
- 如果只是字段名、状态码或 API 标识变化，应优先更新模型或接口 owner，而不是把词汇表扩写成字段清单。
